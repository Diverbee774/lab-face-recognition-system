## 一、项目概述

### 1.1 项目背景
针对高校及科研机构多实验室场景下人员流动性大、管理困难的问题，构建一套基于计算机视觉与微服务架构的智能门禁系统。

### 1.2 系统架构
```
┌─────────────────┐
│   Vue.js 前端   │  管理后台 + 门禁模拟终端
└────────┬────────┘
         │ HTTP / WebSocket
┌────────▼────────┐
│  Java Spring Boot │  业务逻辑、权限控制、数据库
└────────┬────────┘
         │ Thrift RPC
┌────────▼────────┐
│  Python Thrift   │  人脸检测 + 特征提取（纯算法）
└─────────────────┘
```

### 1.3 技术栈
| 层级 | 技术 |
|------|------|
| 前端 | Vue.js + Element UI |
| 后端 | Java Spring Boot |
| 算法服务 | Python + face_recognition + dlib |
| RPC框架 | Apache Thrift |
| 数据库 | MySQL |
| 人脸识别库 | dlib + face_recognition |

### 1.4 职责划分

| 端 | 职责 |
|---|---|
| **Python** | 纯算法服务，接收图片字节流，返回人脸位置+128维向量 |
| **Java** | 业务逻辑、数据库管理、人脸比对算法、对外接口 |

## 三、接口定义 (Thrift)

### 3.1 人脸识别服务 (face.thrift)
```thrift
namespace py face_service

struct FaceInfo {
    1: list<double> encoding,    // 128维人脸向量
    2: i32 top,                   // 人脸框上边界
    3: i32 right,                 // 人脸框右边界
    4: i32 bottom,                // 人脸框下边界
    5: i32 left                   // 人脸框左边界
}

struct EncodeRequest {
    1: required binary image_data  // 图片字节流
}

struct EncodeResponse {
    1: required i32 code,
    2: optional list<FaceInfo> faces,  // 检测到的所有人脸
    3: required string msg
}

service FaceRecognition {
    // 编码接口：接收图片，返回所有人脸的位置+向量
    EncodeResponse encode(1: EncodeRequest req)
}
```

### 3.2 接口说明
| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| encode | image_data (bytes) | code, faces[], msg | 人脸检测+特征提取 |

### 3.3 返回示例
```json
// 成功检测到1个人脸
{
  "code": 200,
  "faces": [{
    "encoding": [0.123, -0.456, ...],  // 128维向量
    "top": 120,
    "right": 280,
    "bottom": 220,
    "left": 180
  }],
  "msg": "success"
}

// 未检测到人脸
{
  "code": 200,
  "faces": [],
  "msg": "No face found"
}
```

## 四、数据库表结构

### 4.1 tb_user (用户表)
| 字段 | 类型 | 说明 |
|------|------|------|
| uid | VARCHAR(20) | 学号/工号 (主键) |
| name | VARCHAR(50) | 姓名 |
| password | VARCHAR(128) | 密码 |
| status | TINYINT | 状态 (1启用/0禁用) |
| has_face | TINYINT | 是否已录入人脸 |
| reg_time | DATETIME | 注册时间 |

### 4.2 tb_lab (实验室表)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR(100) | 实验室名称 |
| location | VARCHAR(100) | 位置 |
| capacity | INT | 最大容量 |
| status | TINYINT | 状态 (1开放/0维护) |
| mode | VARCHAR(20) | 访问模式 (PUBLIC/PRIVATE) |

### 4.3 tb_face_base (人脸底库表)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| uid | VARCHAR(20) | 用户ID |
| lab_id | VARCHAR(64) | 实验室ID（为空表示全局底库） |
| encoding | BLOB | 128维人脸向量 |
| reg_time | DATETIME | 注册时间 |

### 4.4 tb_lab_user_relation (白名单关系表)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| lab_id | INT | 实验室ID |
| uid | VARCHAR(20) | 用户ID |
| add_time | DATETIME | 添加时间 |

### 4.5 tb_access_log (通行日志表)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| uid | VARCHAR(20) | 用户ID |
| lab_id | INT | 实验室ID |
| snapshot_path | VARCHAR(200) | 抓拍照片路径 |
| result | VARCHAR(20) | 通行结果 |
| fail_reason | VARCHAR(100) | 失败原因 |
| access_time | DATETIME | 通行时间 |
